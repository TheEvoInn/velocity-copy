┌──────────────────────────────────────────────────────────────────────────────┐
│ HOW IT WORKS                                                                 │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│ 1. USER UPDATES A SETTING                                                   │
│    └─ Clicks button to change Autopilot mode to "scheduled"                 │
│                                                                              │
│ 2. FRONTEND CAPTURES CHANGE                                                 │
│    └─ UserPreferencesPanel component detects update                         │
│    └─ usePersistentUserData hook invoked                                    │
│                                                                              │
│ 3. PERSISTENCE LAYER SAVES DATA                                             │
│    └─ userDataPersistenceManager function called                            │
│    └─ Old value logged for audit trail                                      │
│    └─ New value stored in UserDataStore                                     │
│    └─ SHA-256 checksum calculated                                           │
│    └─ Modification log updated                                              │
│    └─ Timestamps recorded                                                   │
│                                                                              │
│ 4. AUDIT TRAIL CREATED                                                      │
│    └─ UserDataAuditLog entry created                                        │
│    └─ Explicit user consent flagged (true)                                  │
│    └─ Source recorded as "user_action"                                      │
│    └─ Timestamp logged                                                      │
│                                                                              │
│ 5. UI REFRESHED                                                             │
│    └─ React Query cache invalidated                                         │
│    └─ Components re-fetch latest data                                       │
│    └─ UI updates with new value                                             │
│    └─ User sees confirmation                                                │
│                                                                              │
│ 6. AUTOPILOT USES THE SETTING                                               │
│    └─ Before next cycle, autopilotPersistenceSync called                    │
│    └─ Loads user's saved preferences from database                          │
│    └─ Executes with "scheduled" mode as configured                          │
│    └─ Audit log shows system_access entry                                   │
│                                                                              │
│ 7. DATA SURVIVES EVERYTHING                                                 │
│    └─ Page reload? → Still in database ✅                                   │
│    └─ UI redesign? → Preferences independent ✅                             │
│    └─ System update? → Version-tracked ✅                                   │
│    └─ Autopilot reset? → Config preserved ✅                                │
│    └─ Identity switch? → Settings maintained ✅                             │
│                                                                              │
